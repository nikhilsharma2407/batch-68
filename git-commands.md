1. git init - initialise a repo
2. git status - check the repo status
3. git add <filename> <filename2>...<filenameN>  -> git add index.html git-commands.md  - start tracking the resp file
4. git add . - add all files to staging
5. git reset <filename> or . - unstage the changes
6. git commit -m <message> - commit the staged files
7. git checkout <filename> or . - undo the changes since last commit or staging
8. git checkout -b <branchname>  create a new branch
9. git checkout <branchname>  switch to branch
